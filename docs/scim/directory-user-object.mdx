# Directory User Object

<Subtitle>Discover the Directory User Attributes from the Directory Providers</Subtitle>

Organizations rely on Directory Providers as the source of truth for user information. By setting up SCIM Provisioning,
your app receives real-time updates, ensuring it remains current with this source. Scalekit normalizes response from
Directory Providers that adhere to the SCIM protocol for straightforward easy of implementation.

Your app can [register a Webhook endpoint](docs/scim/setup-directory-events.mdx) to listen to these directory events.
See the [Events Reference](/apis#tag/Webhooks) for more information. Each Event Object has a Directory User Object as
part of its payload.

## Directory User Object Attributes

Each Directory User Object has the following attributes:

| <SimpleCode>id </SimpleCode>                 | A unique identifier for the user within the directory.                        |
| <SimpleCode>organization_id</SimpleCode>     | A unique identifier for the organization to which the user belongs.           |
| <SimpleCode>dp_id</SimpleCode>               | An external identifier for the user from the SCIM directory provider.         |
| <SimpleCode>preferred_username</SimpleCode>  | The preferred username for the user, typically used for login purposes.       |
| <SimpleCode>email </SimpleCode>              | The user's email address.                                                     |
| <SimpleCode>active</SimpleCode>              | Indicates whether the user's account is active.                               |
| <SimpleCode>name</SimpleCode>                | The full name of the user.                                                    |
| <SimpleCode>roles</SimpleCode>               | The role assigned to the user within the organization.                        |
| <SimpleCode>groups.id</SimpleCode>           | A unique identifier for the group to which the user belongs.                  |
| <SimpleCode>groups.name</SimpleCode>         | The name of the group to which the user belongs.                              |
| <SimpleCode>given_name</SimpleCode>          | The user's given name or first name.                                          |
| <SimpleCode>family_name</SimpleCode>         | The user's family name or last name.                                          |
| <SimpleCode>nickname</SimpleCode>            | A nickname or informal name for the user.                                     |
| <SimpleCode>picture</SimpleCode>             | A URL to the user's profile picture.                                          |
| <SimpleCode>phone_number</SimpleCode>        | The user's phone number, formatted with the country code.                     |

| <SimpleCode>address.formatted</SimpleCode>   | The full formatted address of the user.                                       |

| <SimpleCode>raw_attributes</SimpleCode>      | The original payload data from the directory provider                         |

## Custom Attributes

Custom attributes are admin-defined data points or properties that your app can use to tailor and enhance user
experiences. These attributes allow for the personalization of content, features, and functionalities based on specific
user characteristics or preferences. For example, if your B2B SaaS app offers analytics dashboards, you can use a custom
attribute like <SimpleCode>department</SimpleCode> to display relevant dashboards to users.

[Learn more about mapping the necessary attributes from the directory provider](docs/scim/map-user-attributes.mdx).

## Raw Attributes

Scalekit receives information via the SCIM protocol and retains all properties from the directory provider under
the <SimpleCode>raw_attributes</SimpleCode> property without modification.
